Stefanini Developer UP - Rent A Book Sistema de Empréstimos de Livros

Enunciado do desafio Você foi contratado para desenvolver um sistema de empréstimo de livro para uma biblioteca, esse sistema já tinha sido iniciado por outro desenvolvedor que já tinha feito Listar Clientes tanto no frontend quanto no backend, foi feito também um componente de mensagens, use ele para gerar mensagens de alerta, erro e sucesso! Você deve desenvolver as seguintes funcionalidades respeitando as regras, fazer o CRUD (Cadastrar, Listar, Atualizar e Deletar) do Autor, o CRUD de Livros, o CRUD de cliente. Deve também ser feita uma tela para que seja possível emprestar um livro, também deve ser possível listar todos os empréstimos, nessa tabela deve existir uma coluna chamada ações, que vai exibir um botão chamado “Devolver” este botão deve fazer a devolução sem sair da página e atualizar a listagem.

CRUD de Autor

  1. Campos Obrigatórios:
    1. Nome
    2. ISNI
    3. E-mail
    4. Data de Nascimento
    5. Biografia
  2. Regras:
    1. É permitido o Cadastro de apenas um autor por código ISNI
    2. Validar se o e-mail tem pelo menos um ponto e apenas um @ ex: autor@livro.abc
    3. Permitir Apenas Data de nascimento passadas, não podendo conter a data atual nem futura.
    4. Não permitir mais que 200 caracteres na Biografia.
    5. Não permitir mais que 50 caracteres no Nome.

CRUD de Livros

  1. Campos Obrigatórios:
    1. Nome
    2. Autor (um a partir dos autores cadastrados)
    3. Ano de Publicação
    4. Editora
    5. Código ISBN (validar em API externa)
    6. Quantidade de exemplares
  2. Regras:
    1. É permitido o Cadastro de apenas um livro por código ISBN
    2. Apenas um autor por livro
    3. Não permitir mais que 50 caracteres no Nome
    4. Não permitir mais que 50 caracteres na Editora
    5. Permitir o Cadastro somente do ano Atual ou passado em Ano de Publicação
    6. Permitir apenas valores positivos em Quantidade de exemplares

CRUD de cliente

  1. Campos Obrigatórios:
    1. Nome
    2. E-mail
    3. Telefone
  2. Regras:
    1. É permitido o Cadastro de apenas um cliente por e-mail
    2. Validar se o e-mail tem pelo menos um ponto e apenas um @ ex: autor@livro.abc
    3. Não permitir mais que 50 caracteres no Nome
    4. Usar componente de formatação no campo Telefone com o padrão (XX) XXX – XXX – XXX

Cadastro de empréstimos

  1. Campos Obrigatórios:
    1. Livro
    2. Cliente
    3. Data de início
  2. Regras:
    1. Permitir a busca do cliente por parte do nome ou e-mail completo
    2. Permitir a busca do livro por parte do nome do livro ou código ISBN completo
    3. É permitido o empréstimo de somente um exemplar do mesmo título
    4. O cliente pode fazer no máximo 3 empréstimos sem ter que devolver um antes
    5. Validar se um título já teve todos os seus exemplares emprestados antes de emprestar um
    6. Registrar a Data de entrega apenas quando o título for devolvido

Observação:

Usar qualquer API externa para validar o ISBN do livro cadastrado, API sugerida: https://openlibrary.org/dev/docs/api/books Exemplo de uso: https://openlibrary.org/books/OL7353617M.json Use a data de início e data de entrega para determinar se um livro se encontra emprestado ou não. Use também essa informação aliada da quantidade de exemplares para saber se todos os livros estão emprestados.
